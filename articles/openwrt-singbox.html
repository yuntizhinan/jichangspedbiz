Sing-box在OpenWrt软路由系统下的配置与高吞吐专线分流教程

所属版块: 科普专栏

便宜机场推荐：10元左右高性价比机场首选

所属版块: 便宜机场推荐

虽然 Clash 依然是全屋软路由技术的核心霸主，但在 2026 年，一个更加纯粹、速度更快且对硬件运行内存消耗极低的新一代代理内核——Sing-box 正在迅速崛起。在许多 OpenWrt 系统上，使用 Sing-box 作为底层代理，CPU 占用率和发热量只有 Clash 的一半不到，非常适合低配置家用主路由部署。本篇为您带来其配置分流教程。

一、Sing-box 的优势与架构特点

Sing-box 原生支持所有最新协议，包括 VLESS、Reality、Hysteria2 与 Tuic v5。由于它是使用 Go 语言全新编写的精简底座，抛弃了 Clash 许多繁重的历史兼容代码，在软路由上进行千兆 IPLC/IEPL 专线转发时，能够实现近乎零延迟的封包转发效率。支持 极连云（全IEPL） 和 速界 等高端机场的底层导入。

在 速界 后台，点击“复制 Sing-box 订阅”获取 JSON 格式的配置地址。进入软路由插件的后台设置，在订阅管理处粘贴该 JSON 地址，点击“拉取同步”。Sing-box 的分流规则依靠 route.rules 来维护，由于其规则解析极其轻量，建议开启全局 DNS 劫持并启用 dns-direct 机制，这样可以保证国内应用（如微信、支付宝）在直连时不带任何延迟感，而国外 Google 搜索或 Netflix 4K 则能以最大的专线物理宽带速度起飞。
